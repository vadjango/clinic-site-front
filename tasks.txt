створіть форму (після погодження її з викладачем), яка містить наступні

- поля введення даних;
- пояснюючі написи до полів вводу;
- кнопку для підтвердження завершення введення даних;

3.4.2 Поля, які є обов'язковими для введення, необхідно якимось чином
позначити. Якщо таких полів більше, ніж необов'язкових, то помітити краще
необов'язкові поля.
3.4.3 Для тих полів, де можливо, забезпечте вибір значень зі списку або
інші способи вибору значень замість ручного введення.
3.4.4 Для події натискання кнопки підтвердження завершення введення
необхідно реалізувати перевірку на заповненість обов'язкових полів та перевірку
на правильність введених даних (по можливості) засобами HTML та браузера.
3.4.5 Для одного з полів, для якого перевірку допущення помилок
неможливо реалізувати засобами HTML та браузера, виконайте перевірку на
правильність тоді, коли елемент (поле введення) втратить фокус. При виявленні
помилок введення, необхідно після поля з помилкою вставити текст, що
сповіщає про помилку і має колір, відмінний від кольору тексту пояснювальних
написів до полів.
3.4.7 Для події натискання кнопки очищення полів необхідно реалізувати
очистку від введених даних усіх полів форми.
3.4.8 За подією надсилання даних форми при відсутності помилок введення
даних, необхідно відобразити спливаючий блок з текстом, що повідомляє про те,
що введені дані прийняті в обробку. Спливаючий блок повинен мати символ
закриття (х), при кліці по якому він має зникнути.
3.4.9 Продемонструйте результати роботи викладача в браузері.
3.4.10 Складіть звіт, оформіть його відповідно до ДСТУ 3008: 2015 [4] та
захистите роботу.
3.4.11 Звіт повинен містити:
- найменування інформаційної системи;
- HTML-код, CSS-код і JS-код сторінки сайту, що містить форму;
- скріншоти вікна браузера, що відображають вид сторінки до введення
даних і з реакцією програми на помилки або відсутність даних у
обов'язкових полях, на відправку правильних даних.


Валидация происходит в случае, когда пользователь убирает фокус с поля. Если поле пустое, то появляется подпись "Поле не може бути порожнім".
Далее разные поля валидируются по-своему:
    - поля фамилии, имени, отчества, улицы - должны быть только буквы

Если поле не прошло HTML-валидацию и родительский узел поля (т.е. элемент .inner-block) не
содержит errorText, то добавляем этот блок в иннер-бокс. Иначе, если поле прошло валидацию,
то убираем блок errText.

При нажатии на кнопку "Записатися" должно появиться модальное
окно. Для этого нужно в form-wrapper вставить секцию с
самим окном, у которого свойство display = none по умолчанию.
